1. Sebuah No IP 192.168.1.1 jika no IP tersebut dibaca oleh komputer dalam bentuk biner, maka hasil konversi no IP ke dalam bentuk biner...


Jawaban:

Jika no IP 192.168.1.1 dibaca oleh komputer dalam bentuk biner, maka hasil konversinya adalah 11000000.10101000.00000001.00000001. Setiap angka dalam no IP tersebut akan dikonversi ke dalam bentuk biner dengan cara mengkonversi setiap angka yang ada ke dalam bentuk biner. Sebagai contoh, angka 192 akan dikonversi menjadi 11000000, angka 168 akan dikonversi menjadi 10101000, dan seterusnya.


2. Sebuah No IP 11.12.13.14 /8 tentukan network ID pada kelas no IP tersebut


Jawaban:

Kelas no IP tersebut adalah kelas B, dan network ID-nya adalah 11.0.0.0/8. Network ID adalah bagian pertama dari sebuah no IP yang menunjukkan jaringan di mana komputer tersebut berada. Pada kelas B, network ID terdiri dari dua blok pertama dari no IP, yaitu 11 dan 0. Jadi, network ID pada no IP 11.12.13.14/8 adalah 11.0.0.0/8.

15. Layanan yang secara otomatis memberikan no ip kepada komputer adalah


Jawaban:

Layanan yang secara otomatis memberikan nomor IP (Internet Protocol) kepada komputer adalah layanan DHCP (Dynamic Host Configuration Protocol).

Penjelasan:

DHCP adalah protokol jaringan yang digunakan untuk memberikan konfigurasi jaringan kepada komputer yang terhubung ke jaringan, termasuk memberikan nomor IP yang unik kepada setiap komputer. Saat komputer terhubung ke jaringan yang menggunakan DHCP, ia akan mengirim permintaan ke server DHCP untuk mendapatkan konfigurasi jaringan, termasuk nomor IP. Server DHCP akan memberikan nomor IP yang tersedia kepada komputer tersebut, serta informasi lain seperti alamat subnet mask, alamat gateway default, dan alamat DNS (Domain Name System) yang digunakan untuk mengkonversi nama domain menjadi nomor IP.

Nomor IP yang diberikan oleh DHCP dapat digunakan untuk mengidentifikasi komputer yang terhubung ke jaringan, serta menentukan lokasi komputer tersebut dalam jaringan. Dengan menggunakan DHCP, Anda tidak perlu mengonfigurasi nomor IP secara manual untuk setiap komputer yang terhubung ke jaringan, sehingga proses konfigurasi jaringan menjadi lebih mudah.

26. jika Ip 192.168.18.177 brp ip subnet dan ip broad cast


Jika IP tersebut memiliki Mask 255.255.255.0 / Class C Full
maka IP Subnet adalah 192.168.18.0 dan IP Broadcast 192.168.18.255Cara menghitung IP Subnet (IP Network):
Ubah IP menjadi biner:
11000000.10101000.00010010.10110001
Tidak ada suffix (Nilai CIDR)? Gunakan default netmask: /24 yaitu 255.255.255.0.
Ubah Subnet mask tsb menjadi biner juga:
11111111.11111111.11111111.00000000

Perhitungan Network:
Subnet mask AND IP:
[tex]11111111.11111111.11111111.00000000[/tex]
[tex]11000000.10101000.00010010.10110001[/tex]
-------------------------------------------------------- AND
[tex]11000000.10101000.00010010.00000000[/tex] ATAU 192.168.18.0

Perhitungan Broadcast:
Invers Subnet Mask OR Network
[tex]00000000.00000000.00000000.11111111[/tex]
[tex]11000000.10101000.00010010.00000000[/tex]
------------------------------------------------------------------- OR
[tex]11000000.10101000.00010010.11111111[/tex] ATAU 192.168.18.255

Maka dengan IP 192.168.18.177, 
subnet masknya adalah 255.255.255.0

Networknya adalah 192.168.18.0 dengan broadcast 192.168.18.255
